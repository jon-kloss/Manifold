7dThe living-blueprint principles — deliverable #4, final
Interaction principles
01 The map is truth. Every surface is a layer on it or a zoom level of it — never a separate place. Position is never lost.
02 Solver output is always editable. A solve is a starting point; every number it writes can be grabbed and dragged, and it re-solves around you.
03 Nothing mutates without review. Solver, advisor, chat, save re-import — all changes arrive as proposals with provenance; accept is one undo step.
04 ◇ ◈ ◆ everywhere. Planned, under-construction, built read identically on map, graph, and tables — stroke + fill + glyph, never color alone.
05 Numbers change, geometry doesn't. Solves never move cards or pins. Layout is the user's; data is the solver's.
06 No dead ends. Infeasible goals return best-achievable + relaxations; parse failures degrade to manual; AI offline degrades to local heuristics.
07 Orange is a verb. Signal marks what you can do or must see — never decoration. Status belongs to Flow colors, planned-ness to Blueprint blue.
08 Silence is a feature. The advisor earns attention with gated, provenance-backed cards; a badge count is its loudest voice.

6cAI trust rules — offline posture, gating, provenance
AI layer rules
Strictly additive
No solver feature ever depends on AI. Offline / no API key: FEED and CHAT tabs show a quiet AI OFFLINE state; local heuristic warnings (deficits, conflicts, saturation) keep flowing into the same feed — they're computed locally anyway.
Ambient gating — silence is a feature
Local heuristics gate every call: only meaningful events (new deficit, conflict, saturation crossing 75%, power margin swing) arm the advisor; a 30s debounce collapses edit bursts; a visible hourly call budget caps it. Dismissing a card offers "mute this rule" — the gate learns per rule, not globally.
Provenance on everything
Every AI output carries: context scope, snapshot time, state hash, latency (6a SAW line — same grammar as solver proposals in 3a). "▸ view" on the context bar shows exactly the JSON sent. Images are only ever sent in the style-guide flow, explicitly.
One door to change anything
Chat and advisor cannot edit state. Every actionable suggestion routes through proposal review — one system (3c), one muscle memory, zero surprise mutations.

5cWizard spec — entry points, step 3, infeasibility
Wizard behavior
Entry points — all pre-fill the goal
P anywhere · PLAN SUPPLY CHAIN button (2a) · FIX WITH SOLVER on any deficit/saturation row (2d) · advisor CREATE PROPOSAL (2c) · right-click any item icon → "plan production".
Step 3 = proposal review
The wizard's final step IS 3a — same surface, wizard breadcrumb in the banner. RE-SOLVE… returns here to step 1 with constraints intact; each run increments the draft number, previous drafts stay in the archive for comparison.
Infeasible ≠ dead end
If the goal can't be met inside constraints, the solver returns the best achievable rate (e.g. 6.2/8.0) plus the binding constraint, and offers one-tap relaxations: "allow 1 more node claim → 8.0 ✓" · "allow drones → 7.4". Each relaxation shows its cost before re-solving. Never a bare error.
Honest progress
Phases + live log are real solver output, not theater — power users learn why the solver chose what it chose, which is what makes them trust step 3. Cancel is instant and stateless; background solves never interrupt — they wait in the status bar.

4cInteraction spec — solver contract, direct manipulation, keyboard
Factory view interactions
The solver contract
· Every slider drag, recipe swap, clock or tier change re-solves the subgraph live (<5ms budget; solve time always visible).
· Numbers change, geometry doesn't. A solve never moves, adds, or removes cards — structural changes are only ever made by you, or arrive as a proposal.
· Changed values settle-flash amber→ink over 300ms so you can see what the solve touched. Reduced-motion: instant swap.
· Infeasible input (target above input ceiling) doesn't error — the slider hard-stops at the ceiling marker and the binding constraint highlights red (coal feed in 4a).
Direct manipulation
Drag card = move, 16px grid snap. Drag from a port = draw belt; tier picker on drop; invalid targets dim. Alt-drag a bank = split it. Click edge = belt inspector (tier, contents, both endpoints). ⌫ deletes planned entities only — built ones ask "plan a demolition?". Double-click canvas = add machine group (icon-first picker).
Keyboard
R recipes · T target rate · C clock · B belt tier · / find in factory · F frame · ⌘Z undo (solves included) · ESC zoom back to world
Built vs planned inside one factory
A factory is usually mixed: built banks + planned deltas coexist on the same canvas (ghost sub-cards, 4a). DIFF overlay isolates the deltas; a "build list" per factory exports what's left to build in-game.

3cProposal lifecycle & rules — applies to wizard and advisor alike
Lifecycle
DRAFT→ REVIEWING→ ACCEPTED → ◇ PLAN or REJECTED · archived
Invariants
· Proposals never auto-apply and never touch the built layer — accept creates ◇ planned entities only.
· Accept (full or partial) is one undo step. Rejected proposals are archived and recallable.
· Partial accept: excluded items stay in the draft; consequences of exclusion are recomputed live and shown before accept (3a warning strip).
· Every proposal carries provenance: source (solver / advisor / chat), state snapshot time, input hash — so a result is reproducible and auditable.
· If the plan changes while a proposal is open, it gets a STALE badge + one-click RE-SOLVE with the same goal.
· Any item is editable before accept (E) — solver output is a starting point, not a verdict.
Keyboard
↑↓ walk items (map pans along) · SPACE include/exclude · E edit · D diff detail · ⏎ accept included · ESC exit review (draft kept)

Iconography
20 · 28 · 40 — extracted game icons
Game icons are the nouns: never recolored, never redrawn, always on a steel-800 chip. UI verbs (pan, filter, settings) are 1.5px-stroke line glyphs, ink-300, one geometric set. An item is never referred to by text alone where an icon can sit first.
Motion
Fast and mechanical: 120ms panels/hover, 200ms drawers, cubic-bezier(.2,0,0,1) — nothing bouncy. Live-solve updates numbers in place with a 300ms amber→ink settle flash; layout never shifts during a solve. Continuous zoom map→graph is the one cinematic moment (~400ms). Reduced-motion: crossfades replace zoom/drawers, belt animation off, settle flash becomes instant swap.
